Reporte Radiografia por Email

Como deberia generarse y enviarse el reporte HTML con graficos?

A

HTML puro con graficos CSS/SVG inline

Barras de allocation con CSS puro, donuts con SVG inline, tablas HTML estilizadas. Maximo compatibilidad con clientes de correo (Gmail, Outlook, Apple Mail). Sin dependencias externas.

Recomendado — los clientes de email no ejecutan JavaScript ni cargan recursos externos de forma confiable
B

HTML con imagenes de graficos (server-rendered)

Generar los graficos como PNG en el servidor (usando canvas/sharp), embedirlos como base64 en el HTML. Graficos mas sofisticados (treemaps, donuts con gradientes) pero mas complejo de mantener.

C

Link a version web + resumen en email

Email con un resumen liviano (texto + barras CSS) y un boton "Ver Reporte Completo" que abre la radiografia en el navegador (requiere login). Menos trabajo pero depende de que el cliente haga click.

Contexto tecnico

  • Ya tienes Resend configurado para enviar emails HTML
  • Los graficos de Recharts (React) NO funcionan en emails (requieren JS)
  • Gmail elimina <style> tags — hay que usar estilos inline
  • Outlook no soporta SVG en algunos casos — pero si en versiones modernas
  • El reporte se genera desde la data de /api/portfolio/radiografia
