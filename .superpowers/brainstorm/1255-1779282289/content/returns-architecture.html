Retornos por Tipo de Activo - Arquitectura

Como se calcula y desglosa el retorno segun el tipo de activo en la cartera

Fondos Mutuos / ETFs

Retorno Precio
(precioActual / precioCompra - 1) x 100
Dividendos (opcional)
+ dividendos recibidos / costoBase
Retorno Total
= retornoPrecio + retornoDividendos

Acciones (Stocks)

Retorno Precio
(precioActual / precioCompra - 1) x 100
Dividendos (opcional)
+ dividendos recibidos / costoBase
Retorno Total
= retornoPrecio + retornoDividendos

Renta Fija (Bonos)

Devengo (Accrued Interest)
cupon x diasDesdeUltimoPago / 360
30/360 day count convention
Dif. Precio Mercado
(precioMercado - precioCompra) x nominal
Cupones Cobrados (opcional)
+ cuponesPagados
Retorno Total
= devengo + difPrecio + cupones
Nota: Fondos y Acciones comparten la misma logica base (precio). Renta Fija tiene 3 componentes separados. La pregunta es como mostrarlo en la tabla.

Como organizamos la tabla de retornos?

Elige el enfoque que prefieras

A

Tabla unica adaptativa

Una sola tabla que cambia las columnas segun los tipos de activos presentes. Si hay bonos, agrega columnas de devengo y dif. precio. Si solo fondos, muestra la version simple.

Pros

  • Vista compacta, todo junto
  • Facil comparar peso relativo entre clases

Cons

  • Muchas columnas si hay todos los tipos
  • Columnas de bonos vacias para fondos
B

Secciones separadas por clase

Dividir la tabla en secciones: "Renta Variable" (fondos, ETFs, acciones), "Renta Fija" (bonos), "Cash". Cada seccion con columnas optimizadas para su tipo.

Pros

  • Columnas especificas por tipo (devengo solo en RF)
  • Mas legible, sin columnas vacias

Cons

  • Mas largo verticalmente
  • Subtotales por seccion necesarios
C

Hibrido: tabla comun + expandible

Tabla principal con columnas comunes (nombre, peso, valor, retorno total). Click en un bono expande el desglose (devengo / dif. precio / cupones). Click en accion expande dividendos.

Pros

  • Limpio por defecto, detalle on-demand
  • No desperdicia espacio

Cons

  • Desglose oculto puede pasar desapercibido
  • Mas complejo de implementar
