Dividendos entre cartolas: donde los guardamos?

Alpha Vantage nos da ex_dividend_date + amount por ticker. Filtramos entre fechas de snapshots.

Flujo concreto

Snapshot A (31-Mar) ──── ex-div QQQ 12-Abr $0.65 ──── Snapshot B (30-Abr)
→ QQQ tiene 180 units → dividendo periodo = 180 x $0.65 = $117
→ retornoDividendos = $117 / $85,320 (marketValue inicio) = +0.14%

Donde almacenamos los dividendos fetcheados?

Necesitamos persistir el historico de Alpha Vantage para no re-fetchar cada vez

A

Inline en el snapshot (JSONB)

Al calcular retornos, fetchear dividendos y guardarlos dentro de cada holding en el snapshot. Campo dividends: [{date, amount}] en el JSONB de holdings.

Pros

  • Todo junto, sin tabla nueva
  • Snapshot es auto-contenido

Cons

  • Datos duplicados entre snapshots
  • Dificil consultar dividendos globalmente
B

Tabla dedicada: dividend_history

Tabla normalizada con ticker, ex_date, payment_date, amount, source. Se fetchea una vez, se reutiliza para cualquier cliente que tenga ese ticker.

dividend_history
├─ ticker (PK)
├─ ex_dividend_date (PK)
├─ payment_date
├─ amount
├─ source ('alphavantage')
└─ fetched_at

Pros

  • Un fetch sirve para todos los clientes
  • Consultable: "que dividendos pago QQQ en 2025?"
  • Patron igual a bond_prices

Cons

  • Tabla nueva + migracion
C

Hibrido: tabla + campo resumen en snapshot

Tabla dividend_history como cache global. Al guardar snapshot, se calcula y guarda dividendPeriod: $117 y dividendYield: 0.14% inline en cada holding.

Pros

  • Lo mejor de ambos mundos
  • Snapshot tiene el dato precalculado para display rapido
  • Tabla global evita re-fetch

Cons

  • Mas complejidad
