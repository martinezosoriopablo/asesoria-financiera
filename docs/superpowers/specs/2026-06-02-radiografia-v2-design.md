# Radiografia v2 — Design Spec

## Goal

Rebuild the Radiografia page (`/recomendacion/[clientId]`) into a visual, professional portfolio diagnostic tool. Top-down flow: macro allocation first, then instrument-type-aware breakdown with treemap + detail tables, auto-generated observations, and on-demand Claude narrative analysis.

Must work equally well for 3 client profiles:
1. **Acciones directas** — Individual US stocks (AAPL, MSFT, etc.)
2. **Fondos mutuos** — Chilean FM/FI through AGF or corredora
3. **Mixto** — Combination of stocks, bonds, ETFs, and funds

## Current Problems

1. **Sector classification broken** — All stocks show as "Other" because:
   - The ticker regex `^[A-Z]{1,5}$` in the API is too strict (misses 1-2 char tickers)
   - Stock profiles from AV may not be fetching (env var, timeout, or API issues)
   - No fallback classification when AV fails
2. **Page is visually flat** — Just tables, no visual hierarchy or data density
3. **No instrument-type awareness** — Stocks, funds, and bonds all grouped the same way
4. **Trade suggestions are generic** — No specific amounts, no clear priority flow
5. **No narrative** — Advisor gets data but no professional diagnosis text

## Architecture

### Page structure (top-down flow)

```
/recomendacion/[clientId]

Section 1: Header
  - Client name, risk profile, model profile, comite date
  - Total portfolio value (CLP + USD equivalent)
  - Custodian badges

Section 2: Macro Allocation (RV/RF/Alt/Cash)
  - 4 horizontal bars: actual vs model with delta badges
  - Stacked bar chart showing composition
  - Color: green (<3pp), amber (3-10pp), red (>10pp)

Section 3: Desglose por Tipo de Instrumento
  - Tab selector or auto-sections based on what the client has:
    a) Acciones directas → grouped by SECTOR (Technology, Healthcare, etc.)
    b) Fondos mutuos → grouped by CATEGORIA COMITE (RV USA, RF Chile, etc.)
    c) Bonos → grouped by TIPO (Government, Corporate, EM)
    d) ETFs → grouped by CATEGORIA COMITE (same as funds)
  - Each group shows:
    - Treemap (rectangles proportional to weight, colored by deviation)
    - Detail table below with holdings

Section 4: Observaciones Automaticas
  - Bullet list generated by logic rules (no AI):
    - "Concentracion excesiva en Technology (42% vs 25% modelo)"
    - "Sin exposicion a Renta Fija — modelo sugiere 30%"
    - "3 posiciones representan >50% del portafolio"
    - "Sector Healthcare subponderado, vista OW del comite"

Section 5: Analisis Narrativo (on-demand)
  - Button "Generar Analisis"
  - Calls Claude with portfolio data + comite context
  - Renders a professional paragraph summarizing the diagnosis
  - Uses advisor's configured AI model (Sonnet 4 / Opus 4)
```

### Data flow

```
1. POST /api/portfolio/radiografia (existing, enhanced)
   → Returns: allocation, categories, holdings with sector data, observations

2. Frontend renders sections 1-4 immediately

3. On "Generar Analisis" click:
   POST /api/portfolio/radiografia/narrative
   → Sends: allocation summary, deviations, observations, comite context
   → Returns: narrative paragraph from Claude
```

### Section 2: MacroAllocation (enhanced)

### Section 3: InstrumentBreakdown

**3a. Detection logic (which sub-sections to show):**

```typescript
// Classify each holding into instrument type
type InstrumentType = "stock" | "fund" | "bond" | "etf" | "cash";

function detectInstrumentType(holding): InstrumentType {
  const sid = holding.securityId?.trim() || "";
  // Cash
  if (holding.assetClass === "cash" || /cash|caja|liquidez/i.test(holding.fundName))
    return "cash";
  // Bond (has coupon+maturity or CUSIP pattern)
  if ((holding.couponRate && holding.maturityDate) ||
      /^[A-Z0-9]{9}$/.test(sid) && /\d/.test(sid) && /[A-Z]/.test(sid))
    return "bond";
  // Chilean fund (numeric RUN)
  if (/^\d+$/.test(sid)) return "fund";
  // ETF (known ETF ticker or CFIETF)
  if (/^CFIETF/.test(sid) || ETF_TICKERS.has(sid)) return "etf";
  // Stock (remaining non-numeric securityId)
  if (sid && /^[A-Z]{1,6}$/.test(sid)) return "stock";
  // Default
  return "fund";
}
```

Only show sub-sections that have holdings. If client has 0 bonds, skip bonds section.

**3b. Stocks sub-section: "Acciones por Sector"**

- **Treemap**: Recharts Treemap component
  - Each rectangle = one stock
  - Size = portfolio weight
  - Color = deviation vs sector sleeve (green=aligned, amber=slight, red=big deviation)
  - Label: ticker + weight%
  - Hover tooltip: full name, sector, industry, value USD, weight

- **Detail table below treemap**:
  - Grouped by sector with collapsible rows
  - Columns: Ticker | Nombre | Sector | Industry | Valor USD | Peso% | Sector Peso% | Vista Comite
  - Sector header row shows: sector name, total weight, vs sleeve target, delta badge

- **Sector data source**: `stock_profiles` table via AV OVERVIEW API
  - Cache 30 days
  - Fallback: if AV fails for a ticker, classify as "Sin clasificar" (not "Other")

**3c. Funds sub-section: "Fondos por Categoria"**

- **Treemap**: Rectangles = funds, size = weight, color = deviation vs category target
- **Detail table**:
  - Grouped by comite category (RV USA Large Cap, RF Chile, etc.)
  - Columns: Fondo | Administradora | Categoria | Valor CLP | Peso% | vs Modelo

**3d. Bonds sub-section: "Renta Fija Directa"**

- **Treemap**: Rectangles = bonds, size = weight
  - Color by type: government=blue, corporate=teal, EM=amber
- **Detail table**:
  - Columns: Emisor | Cupon | Vencimiento | Rating | Valor USD | Peso%
  - Grouped by: Government / Corporate / EM Sovereign

**3e. ETFs sub-section: "ETFs"**

- Same structure as funds: treemap + table grouped by comite category
- Shows ETF ticker prominently

### Section 4: ObservacionesAutomaticas

Rules engine that generates bullets from data:

```typescript
interface Observation {
  severity: "alta" | "media" | "info";
  icon: "alert" | "warning" | "info";
  text: string;
}

function generateObservations(data: RadiografiaData): Observation[] {
  const obs: Observation[] = [];

  // 1. Macro allocation gaps
  for (const [role, alloc] of Object.entries(data.allocation)) {
    if (alloc.target > 5 && alloc.actual === 0) {
      obs.push({ severity: "alta", icon: "alert",
        text: `Sin exposicion a ${ROLE_LABELS[role]} — modelo sugiere ${alloc.target.toFixed(0)}%` });
    } else if (Math.abs(alloc.delta) > 10) {
      obs.push({ severity: "alta", icon: "alert",
        text: `${ROLE_LABELS[role]} desviado ${alloc.delta > 0 ? '+' : ''}${alloc.delta.toFixed(1)}pp vs modelo` });
    }
  }

  // 2. Concentration risk (top 3 positions > 50%)
  const sorted = holdings.sort((a, b) => b.weight - a.weight);
  const top3Weight = sorted.slice(0, 3).reduce((s, h) => s + h.weight, 0);
  if (top3Weight > 50) {
    obs.push({ severity: "media", icon: "warning",
      text: `Las 3 mayores posiciones representan ${top3Weight.toFixed(0)}% del portafolio` });
  }

  // 3. Single position > 15%
  for (const h of sorted) {
    if (h.weight > 15) {
      obs.push({ severity: "media", icon: "warning",
        text: `${h.name} representa ${h.weight.toFixed(1)}% — considerar diversificar` });
    }
  }

  // 4. Sector vs comite view mismatches
  for (const sector of sectorBreakdown) {
    if (sector.sleeveVista === "UW" && sector.deltaPp > 5) {
      obs.push({ severity: "media", icon: "warning",
        text: `${sector.sector} sobreponderado +${sector.deltaPp.toFixed(1)}pp, comite recomienda Underweight` });
    }
    if (sector.sleeveVista === "OW" && sector.deltaPp < -5) {
      obs.push({ severity: "info", icon: "info",
        text: `${sector.sector} subponderado, comite recomienda Overweight (oportunidad)` });
    }
  }

  // 5. Low confidence classifications
  const lowConf = holdings.filter(h => h.confidence === "low");
  if (lowConf.length > 0) {
    obs.push({ severity: "info", icon: "info",
      text: `${lowConf.length} posicion(es) clasificadas con confianza baja — revisar manualmente` });
  }

  return obs.sort((a, b) => severityOrder[a.severity] - severityOrder[b.severity]);
}
```

Visual: Cards with left color border (red/amber/blue) and icon.

### Fix stock profile fetching

Current bug: ticker regex `^[A-Z]{1,5}$` misses valid tickers. Fix:
- Broaden to `^[A-Z]{1,6}$` (covers tickers like GOOGL, which is 5 chars, and potential 6-char)
- Add fallback: if AV returns no data, mark as "Sin clasificar" instead of "Other"
- Add assetType-based detection: if `assetType === "stock"` in the holding, always try AV regardless of regex

### New narrative endpoint

`POST /api/portfolio/radiografia/narrative`
- Auth: `requireAdvisor()`
- Rate limit: 5/min (AI calls are expensive)
- Loads advisor's AI model preference from `advisors.ai_model`
- Calls Claude via `lib/ai-usage.ts` tracking
- Returns narrative text

## Treemap Implementation

Use `recharts` Treemap component (already in dependencies):

```typescript
import { Treemap, ResponsiveContainer, Tooltip } from "recharts";

// Data shape for treemap
interface TreemapItem {
  name: string;       // ticker or fund name
  size: number;       // weight percentage (determines rectangle size)
  value: number;      // actual value for tooltip
  color: string;      // deviation-based color
  sector?: string;
  delta?: number;
}

// Color scale for deviations
function deviationColor(deltaPp: number): string {
  if (Math.abs(deltaPp) <= 3) return "#22c55e";  // green-500
  if (Math.abs(deltaPp) <= 10) return "#f59e0b"; // amber-500
  return "#ef4444";                                // red-500
}
```

Custom content renderer for rectangles showing ticker + weight%. Minimum rectangle size threshold to avoid rendering labels on tiny slices.

## Constraints

- **Recharts only** — No new charting dependencies (recharts already in package.json)
- **No tax analysis** unless toggled on (default off for internacional clients)
- **AI narrative is optional** — Page must be fully functional without it
- **Mobile-responsive** — Treemap collapses to a simple bar chart on small screens
- **Stock profiles cached 30 days** — Don't re-fetch from AV unnecessarily
- **Observations are deterministic** — Same data always produces same bullets (no randomness)

## Out of Scope

- PDF export of radiografia
- Historical radiografia comparison
- Client portal view
- Automated rebalancing execution
- Tax impact analysis (future toggle)
